# Trust Model Documentation and Skill Guidance

## Architecture Context

Plans 01 and 02 ship explicit local trust records, content hash detection, changed-extension blocking, and trust/untrust management surfaces. This plan updates public documentation, generated docs mirrors, README summaries, extension SDK guidance, and agent skill workflows so users see the new trust model before validating, testing, reloading, or trusting committed code.

## Implementation

### Overview

Update docs and skills to state that native extensions remain unsandboxed arbitrary code execution, project/team extensions require local per-extension trust records, changed hashes block loading until re-trusted, and package/npm/git install support remains future EXTEND_13B scope.

### Key Decisions

1. Remove stale language saying hash-based trust stores are not shipped.
2. Document `.eforge/extension-trust.json` as local, gitignored trust state. If the implementation chooses a different filename, use the actual filename consistently.
3. Document the hash strategy and its limitation: files imported from outside the extension unit are not covered unless the implementation explicitly hashes them. Tell authors to keep committed extension implementation inside the extension file/directory.
4. Update both Claude Code plugin and Pi skill guidance together. Bump `eforge-plugin/.claude-plugin/plugin.json` because plugin skill text changes; do not bump `packages/pi-eforge/package.json`.

## Scope

### In Scope

- Update extension docs for trust records, hash detection, changed-extension behavior, trust/untrust commands, JSON fields, diagnostics, and no-execution trust decisions.
- Update configuration docs to deprecate or reframe `extensions.trustProjectExtensions` according to the implemented behavior while preserving the existing project-team config stripping warning.
- Update README summary for native extension trust.
- Update extension SDK README runtime-loading section.
- Update Pi and Claude Code `/eforge:extend` skill text to require inspection and user confirmation before trust/validate/test/reload when project/team code is involved.
- Sync public web docs and raw mirrors generated by the docs toolchain.
- Update docs/wiring tests that assert command lists, docs snippets, and no stale hash-store language.

### Out of Scope

- New package manifest conventions.
- npm/git install documentation beyond saying it is deferred to EXTEND_13B.
- New approval UI/state for policy gates.
