# Rename eforge-pi package to pi-eforge

## Architecture Context

The `@eforge-build/eforge-pi` package is the Pi ecosystem extension for eforge. Pi ecosystem convention uses a `pi-` prefix (e.g. `pi-ai`, `pi-tui`, `pi-agent-core`), so the current suffix form is an outlier. This plan performs a pure directory + name rename with coordinated documentation and publish-script updates. No behavior changes.

One string in `packages/engine/src/backends/pi-mcp-bridge.ts` (`` `eforge-pi-${name}` ``) is intentionally left untouched — it is eforge's self-identifier when acting as an MCP client and is unrelated to the Pi extension package name.

## Implementation

### Overview

1. Move the directory `packages/eforge-pi/` to `packages/pi-eforge/` via `git mv` to preserve history.
2. Update the package name in `package.json` from `@eforge-build/eforge-pi` to `@eforge-build/pi-eforge`.
3. Update the publish script's path reference and comment.
4. Update all documentation (READMEs, AGENTS.md, architecture.md) referencing the old path or package name.
5. Regenerate `pnpm-lock.yaml` via `pnpm install`.

### Key Decisions

1. **Use `git mv` for the directory rename** — preserves git history for files in `packages/eforge-pi/`.
2. **Do not touch `pi-mcp-bridge.ts`** — the `eforge-pi-${name}` string is eforge's MCP client self-identifier, not the package name.
3. **Do not touch `CHANGELOG.md` or historical plan artifacts in `eforge/plans/monorepo-restructuring-*`** — per project convention, CHANGELOG is owned by the release flow, and historical plans are immutable artifacts.
4. **Do not touch skill directories** (`packages/eforge-pi/skills/eforge-build/` etc.) beyond the parent directory rename — skill names map to `/eforge:*` commands and are unrelated to the package name.

## Scope

### In Scope
- Directory rename `packages/eforge-pi/` → `packages/pi-eforge/`
- Package name change in `packages/pi-eforge/package.json`
- Path + name updates in `scripts/publish-all.mjs`
- Documentation updates across README.md, AGENTS.md, `packages/pi-eforge/README.md`, `packages/client/README.md`, `docs/architecture.md`
- Lockfile regeneration via `pnpm install`

### Out of Scope
- Publishing a deprecation tombstone for the old npm name (handled manually at release time)
- `CHANGELOG.md` (owned by release flow)
- Historical plan artifacts in `eforge/plans/monorepo-restructuring-*`
- `packages/engine/src/backends/pi-mcp-bridge.ts` (unrelated self-identifier string)
- Skill directory names under `packages/pi-eforge/skills/`
- Publishing the renamed package (post-merge release activity)
